CoderDojo (bitmap vs vector graphics)

This example tries to explain the difference between bitmap and vector graphics/images.

Before we start to have a look at the two formats we have to check how our display/screen works. To display something on your screen, your graphics card expects to get a bitmap. This map is a raster of rectangles each having its own color. This little rectangles are called pixels. The following image (fig1) shows such a bitmap of circle.

bitmaps

A bitmap graphic/image stores the data as a raster of pixels. File formats using bitmaps are for example: bmp, jpg, gif. Because every pixel gets stored, the resulting file can get big even if you changed only a few pixels. To avoid using more disc space than needed many file formats (e.g. jpg) support data compression to make the resulting files smaller. Not every data compression works without losing quality! The higher the compression the higher the loss of quality of your image will be.

Advantages of bitmaps

  • no conversion needed to display it - its already a bitmap

Disadvantages of bitmaps

  • the resulting files can get really big
  • bad quality of the image when you zoom in

vector graphics

A vector graphic/image is a text file containing the description of the image that should get drawn. A commonly used file format for vector graphics is SVG (scalable vector graphic). The image (fig2) below shows a circle drawn as a SVG. Beside the circle you can see the SVG code that is necessary to produce the circle.

 

<?xml version="1.0" standalone="no"?>
<svg width="100px" height="100px" version="1.1" xmlns="http://www.w3.org/2000/svg">
     <circle cy="50px" cx="50px" r="45px" stroke-width="2px" stroke="black" fill="none"/>
</svg>

Advantages of SVGs

  • scaling without loss of quality
  • The style of a SVG can be changed by using CSS (cascading style sheets)
  • a SVG image can get created by a javascript on the fly

Ideas for extensions:
